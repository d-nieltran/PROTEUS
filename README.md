# PROTEUS

**Browser-native virtual screening platform for neglected tropical disease drug targets.**

PROTEUS runs pharmacophore-based virtual screening entirely in the browser using WebGPU compute shaders, real PDB crystallographic data, and ChEMBL compound libraries. No backend required — your GPU does the work.

> ⚠️ **Research prototype.** Scoring is unvalidated against benchmark sets (DUD-E). 3D conformers are approximated from SMILES, not generated by RDKit. Hits should be treated as hypotheses, not predictions.

---

## What It Does

PROTEUS fetches real co-crystal structures from the RCSB Protein Data Bank, extracts pharmacophoric features from crystallographic ligand coordinates, pulls drug-like compounds from ChEMBL, and scores them against the binding pocket using a Gaussian overlap function running on a WebGPU WGSL compute shader (64-thread workgroups). It screens continuously, visualizes results in real time, and persists hits to browser storage.

Three validated drug targets for neglected tropical diseases are included:

| Target | PDB | Ligand | Resolution | Disease | Reference |
|--------|-----|--------|------------|---------|-----------|
| PfDHFR | [1J3I](https://www.rcsb.org/structure/1J3I) | Cycloguanil | 2.33 Å | Malaria | Yuvaniyama et al., 2003 |
| LmPTR1 | [1E92](https://www.rcsb.org/structure/1E92) | Methotrexate | 2.20 Å | Leishmaniasis | Gourley et al., 2001 |
| TcTR | [1BZL](https://www.rcsb.org/structure/1BZL) | Quinacrine mustard | 2.40 Å | Chagas disease | Bond et al., 1999 |

## Scoring Function

The pharmacophore scoring uses Gaussian overlap between compound features and target binding site features:

- **Feature types:** hydrogen bond acceptor, hydrogen bond donor, hydrophobic, positive charge, negative charge
- **Gaussian kernel:** overlap weighted by feature importance and distance
- **Penalties:** Lipinski violation (×0.7), TPSA desolvation, rotatable bond entropy, molecular weight complementarity
- **Hit threshold:** score ≥ 2.0

The WGSL compute shader processes compounds in parallel across 64-thread workgroups. Falls back to CPU scoring if WebGPU is unavailable.

## 3D Visualization

The binding pocket viewer (Three.js) is event-reactive:

- **Idle** — double-shell wireframe pocket gently breathes, energy rings orbit, ambient dust particles drift
- **Screening** — compound particles stream toward the pocket, camera orbits faster, features pulse
- **Hit detected** — target-color flash, camera shake, expanding energy ring, burst particles
- **New record** — gold shockwave, heavy shake, 40-particle burst, dust pushed outward
- **Miss streak** — fog thickens, pocket dims, scene feels "colder" until next hit breaks the streak

## Hardware Detection

PROTEUS detects and reports the client's hardware capabilities:

- Apple Silicon chip inference (M1/M2/M3/M4 family from core count + GPU vendor)
- WebGPU adapter properties (vendor, architecture, buffer limits, workgroup size)
- CPU cores, memory, browser
- NPU detection (flagged as present but browser-inaccessible)
- Live compute utilization (GPU vs CPU time per batch cycle)

## Deploy

PROTEUS is a Vite + React 19 app. Deploy to Cloudflare Pages, Vercel, Netlify, or serve statically.

### Cloudflare Pages

Connect the GitHub repo via the Cloudflare dashboard:

- **Framework preset:** None
- **Build command:** `npm run build`
- **Build output directory:** `dist`

No Workers, no D1, no server-side compute. The free tier covers everything — unlimited bandwidth and requests.

### Local

```bash
git clone https://github.com/d-nieltran/PROTEUS.git
cd PROTEUS
npm install
npm run dev
```

Open in a WebGPU-capable browser (Chrome 113+, Edge 113+, or Safari 18+ with feature flag).

---

## Browser Requirements

- **WebGPU** — Chrome 113+, Edge 113+, Safari 18+ (Technology Preview). Firefox: behind flag.
- Falls back to CPU scoring if WebGPU is unavailable (slower but functional).
- ChEMBL API requires CORS — some corporate networks may block it. Falls back to synthetic compounds.
